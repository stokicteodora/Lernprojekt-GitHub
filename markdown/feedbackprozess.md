# Feedbackprozess

## Ablauf

1. Ein Code wird geschrieben/bearbeitet.
2. Die Änderungen werden auf GitHub hochgeladen.
3. Jemand überprüft die Änderungen und gibt Feedback mündlich oder als Kommentar im Issue.
4. Falls nötig, werden Verbesserungen vorgenommen.
5. Danach wird das Issue in Spalte Done verschoben.
